Code Security Information

Code Security

  • Code has been check by SYNK for weekness and vunerabilities and none were discovered.
  • All code is run directly on the user device and only they know the passwords or pins generated.
  • There are no cookies.
  • There are no urls that access google domains.
  • Password strength tests are done localy.
  • Password leaks are checked on HaveIBeenpwned.
  • Password needing to be checked on HaveIBeenpwned are SHA-1 encrypted, then uploaded via https (SSL tunnel) and the SHA-1 hash is checked against the HaveIBeenpwned hash password database.
  • The password strength calculation is based 200 billion password combinations generated per second (Brute force attack).
  • Large numbers shown use the British and US names.
  • The Pins are randomly generated but ignore pins containing same pairs or numbers, date format and year format.
  • 4 digit pins from the following list are ignored as they are the 20 most common pins. "1234", "0000", "7777", "2000", "2222", "9999", "5555", "1112", "8888", "2001", "1111", "1212", "1004", "4444", "6969", "3333", "6666", "1313", "4321", "1010".

  • The code for this website and the tools can be found on GitHub github.com/AKrawczyk/PasswordTools

  • Password check powered by haveibeenpwned.com
  • Secondary scource to check password strength www.security.org/how-secure-is-my-password
